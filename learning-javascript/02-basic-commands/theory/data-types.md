### ğŸ­ Dynamic Typing
JavaScript is a **dynamically typed** language. This means you don't have to tell the computer that a variable is a number or a text. The language figures it out automatically based on the value you assign.
*   You can even change the type later! (e.g., a variable holding a number can later hold a string).

### ğŸ§± Primitive Types (The Building Blocks)
These are the simplest forms of data. They are immutable (cannot be changed, only replaced).

1.  **`Number`**: Represents both integers and floating-point numbers.
    *   Examples: `42`, `3.14`, `-10`.
    *   **Special Values:**
        *   `Infinity`: Result of dividing by zero (e.g., `1 / 0`).
        *   `NaN` (Not a Number): Result of an invalid math operation (e.g., `"Hello" * 5`).

2.  **`String`**: Text data.
    *   Can be wrapped in single quotes `' '`, double quotes `" "`, or backticks `` ` ` `` (Template Strings).
    *   Example: `"Hello World"`.

3.  **`Boolean`**: Logical entities.
    *   Values: `true` or `false`.
    *   Used for decisions (if/else).

4.  **`Undefined`**:
    *   Represents a variable that has been declared but **has not yet been assigned a value**.
    *   It's JavaScript's way of saying "I don't know what this is yet".

5.  **`Null`**:
    *   Represents an **intentional absence** of any object value.
    *   It's the programmer's way of saying "This is empty on purpose".

    *   Used for integers too large to be represented by the `Number` type.

7.  **`Symbol`**:
    *   Unique and immutable identifiers (advanced usage).

### ğŸ“¦ Reference Types (Objects)
These are more complex structures that can hold collections of data.

1.  **`Object`**:
    *   A collection of key-value pairs.
    *   Example: `{ name: "Neto", age: 25 }`.

2.  **`Array`**:
    *   Technically a special type of **Object** used to store ordered lists.
    *   Example: `[1, 2, 3, "Banana"]`.
